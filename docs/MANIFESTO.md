# The CP Notebook Manifesto

> "We are not building another note-taking application.
> We are building a notebook that grows with its owner."

---

# Why

Competitive Programming is not a programming problem.

It is a learning problem.

Everyone solves problems.

Everyone forgets problems.

Very few people compound knowledge.

This project exists so that every solved problem permanently improves the learner.

---

# Vision

A notebook should behave like a great teacher.

It should remember everything.

It should evolve continuously.

It should know where you struggle.

It should know what you have mastered.

It should know what you are about to forget.

Every interaction should make the notebook better.

---

# Core Beliefs

• Learning > Documentation

• Understanding > Collection

• Evolution > Regeneration

• Memory > Beauty

• Reflection > Consumption

• AI as Teacher > AI as Writer

• Knowledge Graph > Folder Hierarchy

• Visual Thinking > Walls of Text

---

# Every solved problem should improve:

- the notebook

- the learner

- future revision

- future recommendations

- future explanations

Nothing is isolated.

Everything compounds.

---

# One Rule

Before implementing anything ask:

> Will this make the notebook smarter after the next solved problem?

If not,

don't build it.

---

# End Goal

A beginner starts with an empty notebook.

Years later,

the notebook has become

their mentor,

their memory,

their textbook,

their contest companion,

and the history of how they became strong.
